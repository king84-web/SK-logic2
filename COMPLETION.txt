## ğŸ”„ Database Schema

### 3 Models Created

#### ServiceBooking
```
- id: String @id
- name, email, phone, service: String
- message, date: Optional
- status: pending | confirmed | completed | cancelled
- createdAt, updatedAt: DateTime
- Indexes: email, service, status
```

#### ContactMessage
```
- id: String @id
- name, email, subject, message: String
- status: new | read | responded
- createdAt, updatedAt: DateTime
- Indexes: email, status
```

#### AcademyEnrollment
```
- id: String @id
- name, email, phone, course: String
- status: enrolled | completed | cancelled
- createdAt, updatedAt: DateTime
- Indexes: email, course, status
```

## ğŸ”Œ API Endpoints

### 3 POST Endpoints

#### POST /api/bookings
- **Input**: name, email, phone, service, date?, message?
- **Output**: { message, id, booking }
- **Actions**: Save to DB + send emails
- **Emails**: Confirmation to customer, notification to admin

#### POST /api/contact
- **Input**: name, email, subject, message
- **Output**: { message, id }
- **Actions**: Save to DB + send emails
- **Emails**: Confirmation to sender, notification to admin

#### POST /api/academy
- **Input**: name, email, phone, course
- **Output**: { message, id, enrollment }
- **Actions**: Save to DB + send emails
- **Emails**: Welcome to student, notification to admin
